## 2. 模型架构概览

MiniMind 采用标准的 **Decoder-Only Transformer** 架构，主要组件包括：

```
┌─────────────────────────────────────────────────────────┐
│                    MiniMind 架构                         │
├─────────────────────────────────────────────────────────┤
│  输入 tokens → 词嵌入层 (Embedding)                      │
│       ↓                                                  │
│  ┌─────────────────────────────────────────────────┐    │
│  │              Transformer Block × N               │    │
│  │  ┌───────────────────────────────────────────┐  │    │
│  │  │  RMSNorm → 多头注意力 (GQA + RoPE)        │  │    │
│  │  │       ↓                                    │  │    │
│  │  │  + 残差连接                                │  │    │
│  │  │       ↓                                    │  │    │
│  │  │  RMSNorm → FeedForward (SwiGLU / MoE)     │  │    │
│  │  │       ↓                                    │  │    │
│  │  │  + 残差连接                                │  │    │
│  │  └───────────────────────────────────────────┘  │    │
│  └─────────────────────────────────────────────────┘    │
│       ↓                                                  │
│  RMSNorm → 语言模型头 (LM Head) → 输出 logits            │
└─────────────────────────────────────────────────────────┘
```

## 3. 数学公式详解

### 3.1 RMSNorm (均方根归一化)

RMSNorm 替代传统的 LayerNorm，移除均值中心化，仅使用均方根进行归一化：

$$\text{RMSNorm}(x) = \frac{x}{\text{RMS}(x)} \odot \gamma$$

其中：

$$\text{RMS}(x) = \sqrt{\frac{1}{d}\sum_{i=1}^{d}x_i^2 + \epsilon}$$

- $d$: 隐藏层维度
- $\gamma$: 可学习的缩放参数
- $\epsilon = 10^{-5}$: 数值稳定性常数

**代码实现**：
```python
class RMSNorm(torch.nn.Module):
    def __init__(self, dim: int, eps: float = 1e-5):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))

    def _norm(self, x):
        return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)

    def forward(self, x):
        return self.weight * self._norm(x.float()).type_as(x)
```

### 3.2 RoPE (旋转位置编码)

RoPE 通过旋转矩阵为 token 注入相对位置信息：

$$\begin{pmatrix} q_m \\ q_{m+d/2} \end{pmatrix} = \begin{pmatrix} \cos(m\theta) & -\sin(m\theta) \\ \sin(m\theta) & \cos(m\theta) \end{pmatrix} \begin{pmatrix} q_0 \\ q_{d/2} \end{pmatrix}$$

其中：
- $m$: token 位置
- $\theta_i = \text{rope\_theta}^{-2i/d}$: 频率项，rope_theta = 1000000.0

**预计算频率**：
```python
def precompute_freqs_cis(dim: int, end: int = 32768, theta: float = 1e6):
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim))
    t = torch.arange(end, device=freqs.device)
    freqs = torch.outer(t, freqs).float()
    freqs_cos = torch.cat([torch.cos(freqs), torch.cos(freqs)], dim=-1)
    freqs_sin = torch.cat([torch.sin(freqs), torch.sin(freqs)], dim=-1)
    return freqs_cos, freqs_sin
```

**应用旋转嵌入**：
```python
def apply_rotary_pos_emb(q, k, cos, sin, unsqueeze_dim=1):
    def rotate_half(x):
        return torch.cat((-x[..., x.shape[-1] // 2:], x[..., : x.shape[-1] // 2]), dim=-1)
    
    q_embed = (q * cos.unsqueeze(unsqueeze_dim)) + (rotate_half(q) * sin.unsqueeze(unsqueeze_dim))
    k_embed = (k * cos.unsqueeze(unsqueeze_dim)) + (rotate_half(k) * sin.unsqueeze(unsqueeze_dim))
    return q_embed, k_embed
```

### 3.3 GQA (分组查询注意力)

GQA 是 MQA 和 MHA 的折中方案，将查询头分组共享 KV 头：

$$\text{GQA}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**分组关系**：
- 查询头数：$N_q = \text{num\_attention\_heads} = 8$
- KV 头数：$N_{kv} = \text{num\_key\_value\_heads} = 2$
- 每组大小：$N_{rep} = N_q / N_{kv} = 4$

**KV 重复机制**：
```python
def repeat_kv(x: torch.Tensor, n_rep: int) -> torch.Tensor:
    bs, slen, num_key_value_heads, head_dim = x.shape
    if n_rep == 1:
        return x
    return (x[:, :, :, None, :]
            .expand(bs, slen, num_key_value_heads, n_rep, head_dim)
            .reshape(bs, slen, num_key_value_heads * n_rep, head_dim))
```

### 3.4 注意力机制

**缩放点积注意力**：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中 $d_k = \text{head\_dim} = \text{hidden\_size} / \text{num\_attention\_heads}$

**因果掩码**（防止未来信息泄露）：
$$M_{ij} = \begin{cases} 0 & \text{if } i \geq j \\ -\infty & \text{if } i < j \end{cases}$$

**代码实现**：
```python
class Attention(nn.Module):
    def forward(self, x, position_embeddings, past_key_value=None, use_cache=False, attention_mask=None):
        bsz, seq_len, _ = x.shape
        xq, xk, xv = self.q_proj(x), self.k_proj(x), self.v_proj(x)
        xq = xq.view(bsz, seq_len, self.n_local_heads, self.head_dim)
        xk = xk.view(bsz, seq_len, self.n_local_kv_heads, self.head_dim)
        xv = xv.view(bsz, seq_len, self.n_local_kv_heads, self.head_dim)
        
        # 应用 RoPE
        cos, sin = position_embeddings
        xq, xk = apply_rotary_pos_emb(xq, xk, cos[:seq_len], sin[:seq_len])
        
        # KV Cache
        if past_key_value is not None:
            xk = torch.cat([past_key_value[0], xk], dim=1)
            xv = torch.cat([past_key_value[1], xv], dim=1)
        past_kv = (xk, xv) if use_cache else None
        
        # Flash Attention 或 手动实现
        if self.flash and seq_len != 1:
            output = F.scaled_dot_product_attention(xq, xk, xv, is_causal=True)
        else:
            scores = (xq @ xk.transpose(-2, -1)) / math.sqrt(self.head_dim)
            mask = torch.triu(torch.full((seq_len, seq_len), float("-inf")), diagonal=1)
            scores = scores + mask
            scores = F.softmax(scores.float(), dim=-1)
            output = scores @ xv
        
        output = output.transpose(1, 2).reshape(bsz, seq_len, -1)
        return self.o_proj(output), past_kv
```

### 3.5 SwiGLU 激活函数

SwiGLU 是 Swish 和 GLU 的结合：

$$\text{SwiGLU}(x) = \text{Swish}(xW_g) \otimes xW_u$$

$$\text{输出} = \text{down\_proj}(\text{SwiGLU}(x))$$

其中：
- $W_g$: gate 投影 (`gate_proj`)
- $W_u$: up 投影 (`up_proj`)
- $\otimes$: 逐元素乘法
- $\text{Swish}(x) = x \cdot \sigma(\beta x)$，$\beta=1$ 时为 SiLU

**中间层维度**：
$$\text{intermediate\_size} = 64 \times \left\lceil\frac{\text{hidden\_size} \times 8/3}{64}\right\rceil$$

**代码实现**：
```python
class FeedForward(nn.Module):
    def __init__(self, config: MiniMindConfig):
        super().__init__()
        if config.intermediate_size is None:
            intermediate_size = int(config.hidden_size * 8 / 3)
            config.intermediate_size = 64 * ((intermediate_size + 64 - 1) // 64)
        self.gate_proj = nn.Linear(config.hidden_size, config.intermediate_size, bias=False)
        self.down_proj = nn.Linear(config.intermediate_size, config.hidden_size, bias=False)
        self.up_proj = nn.Linear(config.hidden_size, config.intermediate_size, bias=False)
        self.act_fn = ACT2FN[config.hidden_act]  # 'silu'

    def forward(self, x):
        return self.down_proj(self.act_fn(self.gate_proj(x)) * self.up_proj(x))
```

### 3.6 MoE (混合专家)

MoE 通过门控机制动态选择专家网络：

**门控函数**：
$$g(x) = \text{Softmax}(x \cdot W_g)$$

**Top-K 选择**：
$$(\text{topk\_weight}, \text{topk\_idx}) = \text{TopK}(g(x), k)$$

**归一化**：
$$\text{topk\_weight}_i = \frac{\text{topk\_weight}_i}{\sum_j \text{topk\_weight}_j + \epsilon}$$

**专家输出**：
$$y = \sum_{i=1}^{k} \text{topk\_weight}_i \cdot E_{\text{topk\_idx}_i}(x) + \text{shared\_experts}(x)$$

**辅助损失**（负载均衡）：
$$\mathcal{L}_{aux} = \alpha \cdot \sum_{i=1}^{N} f_i \cdot P_i$$

其中：
- $f_i$: 专家 $i$ 被选择的频率
- $P_i$: 专家 $i$ 的平均门控分数
- $\alpha = \text{aux\_loss\_alpha}$

**MoE 配置参数**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `use_moe` | False | 是否启用 MoE |
| `num_experts_per_tok` | 2 | 每个 token 选择的专家数 |
| `n_routed_experts` | 4 | 总专家数量 |
| `n_shared_experts` | 1 | 共享专家数量 |
| `scoring_func` | 'softmax' | 评分函数 |
| `aux_loss_alpha` | 0.1 | 辅助损失权重 |
| `norm_topk_prob` | True | 是否归一化 top-k 概率 |

### 3.7 损失函数

**交叉熵损失**（带 Mask）：
$$\mathcal{L}_{CE} = -\frac{\sum_{i=1}^{T} m_i \cdot \log p(y_i | y_{<i})}{\sum_{i=1}^{T} m_i}$$

其中 $m_i$ 是 loss mask（仅对需要预测的位置计算损失）。

**总损失**（含 MoE 辅助损失）：
$$\mathcal{L}_{total} = \mathcal{L}_{CE} + \mathcal{L}_{aux}$$

**代码实现**：
```python
loss_fct = nn.CrossEntropyLoss(reduction='none')
res = model(X)
loss = loss_fct(res.logits.view(-1, res.logits.size(-1)), Y.view(-1)).view(Y.size())
loss = (loss * loss_mask).sum() / loss_mask.sum()  # masked loss
loss += res.aux_loss  # MoE 辅助损失
```

**DPO 损失**（偏好优化）：
$$\mathcal{L}_{DPO} = -\log \sigma\left(\beta \left[(\log \pi_\theta(y_w|x) - \log \pi_\theta(y_l|x)) - (\log \pi_{ref}(y_w|x) - \log \pi_{ref}(y_l|x))\right]\right)$$

其中：
- $y_w$: 被选择的回复 (chosen)
- $y_l$: 被拒绝的回复 (rejected)
- $\pi_{ref}$: 参考模型（冻结）
- $\beta = 0.1$: 温度参数

---

### 3.8 学习率调度

**余弦退火调度**：
$$\eta_t = \frac{\eta_{max}}{10} + 0.5 \cdot \eta_{max} \cdot \left(1 + \cos\left(\pi \cdot \frac{t}{T}\right)\right)$$

- $\eta_{max}$: 最大学习率
- $t$: 当前步数
- $T$: 总步数

**代码实现**：
```python
def get_lr(current_step, total_steps, lr):
    return lr / 10 + 0.5 * lr * (1 + math.cos(math.pi * current_step / total_steps))
```

## 4. 模型工作原理

### 4.1 前向传播流程

```
1. 输入编码
   └── input_ids (batch_size, seq_len) 
       → Embedding (vocab_size, hidden_size) 
       → hidden_states (batch_size, seq_len, hidden_size)

2. 位置编码
   └── 根据当前序列位置 start_pos:start_pos+seq_len
       从预计算的 freqs_cos/freqs_sin 中提取 RoPE 张量

3. Transformer 层处理（重复 num_hidden_layers 次）
   ├── 输入归一化：RMSNorm(hidden_states)
   ├── 自注意力：Attention(normed_x, RoPE, past_key_value) → attn_output
   ├── 残差连接：hidden_states = hidden_states + attn_output
   ├── 再次归一化：RMSNorm(hidden_states)
   ├── 前馈网络：FeedForward/MoE(normed_x) → ffn_output
   └── 残差连接：hidden_states = hidden_states + ffn_output

4. 输出处理
   ├── 最终归一化：RMSNorm(hidden_states) → last_hidden_state
   └── LM Head：Linear(last_hidden_state) → logits (batch_size, seq_len, vocab_size)
```

**完整前向传播代码**：
```python
class MiniMindForCausalLM(PreTrainedModel, GenerationMixin):
    def forward(self, input_ids, attention_mask=None, past_key_values=None, 
                use_cache=False, logits_to_keep=0, **args):
        # 模型主体前向传播
        h, past_kvs, aux_loss = self.model(
            input_ids=input_ids,
            attention_mask=attention_mask,
            past_key_values=past_key_values,
            use_cache=use_cache,
            **args
        )
        # 获取需要计算 logits 的部分
        slice_indices = slice(-logits_to_keep, None) if isinstance(logits_to_keep, int) else logits_to_keep
        logits = self.lm_head(h[:, slice_indices, :])
        
        return CausalLMOutputWithPast(
            last_hidden_state=h,
            logits=logits,
            aux_loss=aux_loss,
            past_key_values=past_kvs
        )
```

### 4.2 推理流程（自回归生成）

```
初始化：input_ids = [bos_token] + prompt_tokens

for step in range(max_new_tokens):
    1. 将当前序列输入模型：outputs = model(input_ids)
    2. 获取最后一个位置的 logits: next_token_logits = outputs.logits[:, -1, :]
    3. 应用 Temperature 缩放：logits = next_token_logits / temperature
    4. 应用 Top-P 滤波：保留累积概率 > top_p 的 token
    5. 从剩余分布中采样：next_token = multinomial_sample(probs)
    6. 将新 token 追加到序列：input_ids = concat(input_ids, next_token)
    7. 如果遇到 EOS token，停止生成
    8. 如果达到 max_new_tokens，停止生成

返回：生成的完整序列
```

**生成参数**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `temperature` | 0.85 | 采样温度，越高越随机 |
| `top_p` | 0.85 | Nucleus 采样阈值 |
| `max_new_tokens` | 8192 | 最大生成长度 |
| `repetition_penalty` | 1.0 | 重复惩罚系数 |

### 4.3 KV Cache 机制

KV Cache 通过缓存历史 KV 状态加速自回归推理：

**原理**：
- 在自回归生成中，历史 token 的 K 和 V 不会改变
- 缓存历史 KV，避免重复计算
- 每步只需计算当前 token 的 QKV

**代码实现**：
```python
# 在 Attention.forward 中
if past_key_value is not None:
    xk = torch.cat([past_key_value[0], xk], dim=1)  # 拼接历史 K
    xv = torch.cat([past_key_value[1], xv], dim=1)  # 拼接历史 V
past_kv = (xk, xv) if use_cache else None  # 返回给下一轮
```

**性能提升**：
- 无 Cache：每步计算所有 token 的 KV，复杂度 $O(n^2)$
- 有 Cache：每步只计算 1 个 token 的 KV，复杂度 $O(n)$

## 5. 模型交互机制

### 5.1 与数据集的交互

MiniMind 通过 `dataset/lm_dataset.py` 中定义的 Dataset 类进行数据交互：

| Dataset 类 | 用途 | 输入格式 | 输出格式 |
|-----------|------|----------|----------|
| `PretrainDataset` | 预训练 | `{"text": "..."}` | `(X, Y, loss_mask)` |
| `SFTDataset` | 监督微调 | `{"conversations": [...]}` | `(X, Y, loss_mask)` |
| `DPODataset` | 偏好优化 | `{"chosen": [...], "rejected": [...]}` | `dict(chosen/rejected)` |
| `RLAIFDataset` | AI 反馈学习 | `{"conversations": [...]}` | `{"prompt": ..., "answer": ...}` |

**PretrainDataset 数据流**：
```python
class PretrainDataset(Dataset):
    def __getitem__(self, index):
        sample = self.samples[index]
        encoding = self.tokenizer(
            str(sample['text']),
            max_length=self.max_length,
            padding='max_length',
            truncation=True,
            return_tensors='pt'
        )
        input_ids = encoding.input_ids.squeeze()
        loss_mask = (input_ids != self.tokenizer.pad_token_id)
        
        X = torch.tensor(input_ids[:-1], dtype=torch.long)  # [0, n-1]
        Y = torch.tensor(input_ids[1:], dtype=torch.long)   # [1, n]
        loss_mask = torch.tensor(loss_mask[1:], dtype=torch.long)
        return X, Y, loss_mask
```

**SFTDataset Loss Mask 生成**：
```python
def _generate_loss_mask(self, input_ids):
    loss_mask = [0] * len(input_ids)
    i = 0
    while i < len(input_ids):
        # 检测assistant起始标记
        if input_ids[i:i + len(self.bos_id)] == self.bos_id:
            start = i + len(self.bos_id)
            end = start
            # 找到assistant回复的结束位置
            while end < len(input_ids):
                if input_ids[end:end + len(self.eos_id)] == self.eos_id:
                    break
                end += 1
            # 对assistant回复内容设置loss_mask=1
            for j in range(start + 1, min(end + len(self.eos_id) + 1, self.max_length)):
                loss_mask[j] = 1
            i = end + len(self.eos_id) if end < len(input_ids) else len(input_ids)
        else:
            i += 1
    return loss_mask
```

### 5.2 与训练脚本的交互

模型通过以下训练脚本进行不同阶段的训练：

| 训练脚本 | 阶段 | 输入权重 | 输出权重 | 学习率 |
|---------|------|----------|----------|--------|
| `train_pretrain.py` | 预训练 | 随机初始化 | `pretrain_*.pth` | 5e-4 |
| `train_full_sft.py` | 监督微调 | `pretrain_*.pth` | `full_sft_*.pth` | 5e-7 |
| `train_dpo.py` | RLHF-DPO | `full_sft_*.pth` | `rlhf_*.pth` | 1e-8 |
| `train_lora.py` | LoRA 微调 | `full_sft_*.pth` | `lora_*.pth` | 1e-4 |
| `train_distillation.py` | 知识蒸馏 | 教师模型 + 学生模型 | `full_sft_*.pth` | - |
| `train_distill_reason.py` | 推理蒸馏 | 教师模型 + 学生模型 | `reason_*.pth` | - |

**预训练流程** (`train_pretrain.py`)：
```python
def train_epoch(epoch, wandb):
    loss_fct = nn.CrossEntropyLoss(reduction='none')
    for step, (X, Y, loss_mask) in enumerate(train_loader):
        X = X.to(args.device)
        Y = Y.to(args.device)
        loss_mask = loss_mask.to(args.device)
        
        # 学习率调度
        lr = get_lr(epoch * iter_per_epoch + step, 
                    args.epochs * iter_per_epoch, 
                    args.learning_rate)
        for param_group in optimizer.param_groups:
            param_group['lr'] = lr
        
        # 前向传播
        with ctx:  # 混合精度上下文
            res = model(X)
            loss = loss_fct(res.logits.view(-1, res.logits.size(-1)), Y.view(-1)).view(Y.size())
            loss = (loss * loss_mask).sum() / loss_mask.sum()
            loss += res.aux_loss
            loss = loss / args.accumulation_steps  # 梯度累积
        
        # 反向传播
        scaler.scale(loss).backward()
        
        if (step + 1) % args.accumulation_steps == 0:
            scaler.unscale_(optimizer)
            torch.nn.utils.clip_grad_norm_(model.parameters(), args.grad_clip)
            scaler.step(optimizer)
            scaler.update()
            optimizer.zero_grad(set_to_none=True)
```

### 5.3 与推理脚本的交互

`eval_model.py` 负责模型推理：

**模型加载**：
```python
def init_model(args):
    tokenizer = AutoTokenizer.from_pretrained('./model/')
    
    if args.load == 0:  # 原生 PyTorch 权重
        moe_path = '_moe' if args.use_moe else ''
        modes = {0: 'pretrain', 1: 'full_sft', 2: 'rlhf', 3: 'reason', 4: 'grpo'}
        ckp = f'./{args.out_dir}/{modes[args.model_mode]}_{args.hidden_size}{moe_path}.pth'
        
        model = MiniMindForCausalLM(MiniMindConfig(
            hidden_size=args.hidden_size,
            num_hidden_layers=args.num_hidden_layers,
            use_moe=args.use_moe
        ))
        model.load_state_dict(torch.load(ckp, map_location=args.device), strict=True)
        
        # 可选 LoRA 加载
        if args.lora_name != 'None':
            apply_lora(model)
            load_lora(model, f'./{args.out_dir}/lora/{args.lora_name}_{args.hidden_size}.pth')
    else:  # Transformers 格式
        transformers_model_path = './MiniMind2'
        model = AutoModelForCausalLM.from_pretrained(transformers_model_path, trust_remote_code=True)
    
    return model.eval().to(args.device), tokenizer
```

### 6.4 参数量计算公式

**基础参数量**（不含 MoE）：
$$\text{Embedding} = \text{vocab\_size} \times \text{hidden\_size}$$
$$\text{Attention} = \text{num\_layers} \times (4 \times \text{hidden\_size}^2)$$
$$\text{FFN} = \text{num\_layers} \times (3 \times \text{hidden\_size} \times \text{intermediate\_size})$$
$$\text{Norm} = \text{num\_layers} \times 2 \times \text{hidden\_size}$$
$$\text{LM Head} = \text{hidden\_size} \times \text{vocab\_size}$$

**MoE 参数量**：
$$\text{MoE FFN} = \text{num\_layers} \times (\text{n\_routed\_experts} + \text{n\_shared\_experts}) \times (3 \times \text{hidden\_size} \times \text{intermediate\_size})$$
$$\text{Gate} = \text{num\_layers} \times (\text{n\_routed\_experts} \times \text{hidden\_size})$$

**激活参数量**（每 token）：
$$\text{Active Params} = \text{Embedding} + \text{Attention} + \text{Norm} + \text{LM Head} + \text{num\_layers} \times (\text{num\_experts\_per\_tok} + \text{n\_shared\_experts}) \times (3 \times \text{hidden\_size} \times \text{intermediate\_size})$$

## 7. 参数调整指南

### 7.1 训练阶段参数调整

#### 预训练阶段 (Pretrain)

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `learning_rate` | 5e-4 | 基础学习率 |
| `batch_size` | 32 | 批次大小 |
| `max_seq_len` | 512 | 序列长度 |
| `epochs` | 1~2 | 训练轮次 |
| `accumulation_steps` | 8 | 梯度累积步数 |
| `grad_clip` | 1.0 | 梯度裁剪阈值 |

**调整建议**：
- 如果显存不足：减小 `batch_size`，增加 `accumulation_steps`
- 如果训练不稳定：降低 `learning_rate` 至 3e-4 或 1e-4
- 如果序列较长：增加 `max_seq_len` 至 1024 或 2048

#### 监督微调阶段 (SFT)

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `learning_rate` | 5e-7 | 比预训练小 1000 倍 |
| `batch_size` | 16 | 批次大小 |
| `max_seq_len` | 512~2048 | 对话长度 |
| `epochs` | 2~4 | 微调轮次 |

**调整建议**：
- 如果出现过拟合：减少 `epochs` 或增加 `dropout`
- 如果欠拟合：增加 `epochs` 或 `learning_rate`
- 长对话场景：增加 `max_seq_len` 至 2048 或 4096

#### RLHF-DPO 阶段

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `learning_rate` | 1e-8 | 极小学习率，防止遗忘 |
| `batch_size` | 8 | 批次大小 |
| `max_seq_len` | 1024~3000 | 偏好对长度 |
| `epochs` | 1~2 | 微调轮次 |
| `beta` | 0.1 | DPO 温度参数 |

**调整建议**：
- 学习率必须很小，否则会遗忘 SFT 学到的能力
- `beta` 越大，对参考模型的偏离惩罚越大

#### LoRA 微调

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `learning_rate` | 1e-4 | LoRA 学习率 |
| `rank` | 8 | LoRA 秩 |
| `batch_size` | 32 | 批次大小 |
| `epochs` | 5~10 | 微调轮次 |

**调整建议**：
- `rank` 越大，可训练参数越多，但可能过拟合
- 领域适配任务：`rank=8` 或 `rank=16`
- 简单任务：`rank=4` 即可

### 7.2 模型架构参数调整

#### 扩大模型规模

**增加隐藏层维度**：
```python

# 参数量增加约 4 倍
```

**增加层数**：
```python
# Small → Standard

# 参数量增加约 2 倍
```

**增加注意力头数**：
```python
num_attention_heads: 8 → 16
num_key_value_heads: 2 → 4  # 保持 GQA 比例
```

#### 启用 MoE

```python
use_moe = True
num_experts_per_tok = 2  # 每个 token 激活 2 个专家
n_routed_experts = 8     # 总专家数
n_shared_experts = 1     # 共享专家
```

**效果**：
- 总参数量增加，但激活参数量基本不变
- 适合需要大容量但显存有限的场景

#### 调整序列长度

```python
max_position_embeddings: 32768 → 65536  # 支持更长上下文
rope_theta: 1000000.0 → 10000000.0      # 调整 RoPE 频率
```

**注意**：增加序列长度会显著增加显存占用（$O(n^2)$）

### 7.3 推理参数调整

#### Temperature 调整

| temperature | 效果 | 适用场景 |
|-------------|------|----------|
| 0.1 ~ 0.3 | 非常确定，几乎贪婪 | 代码生成、数学计算 |
| 0.5 ~ 0.7 | 平衡 | 通用对话 |
| 0.8 ~ 1.2 | 更随机 | 创意写作 |
| > 1.5 | 高度随机 | 诗歌、艺术创作 |

#### Top-P 调整

| top_p | 效果 | 适用场景 |
|-------|------|----------|
| 0.5 ~ 0.7 | 保守，仅从高概率 token 采样 | 事实性问答 |
| 0.8 ~ 0.9 | 平衡 | 通用对话 |
| 0.95 ~ 1.0 | 开放，几乎不限制 | 创意写作 |

#### 最大生成长度

```python
max_new_tokens: 512      # 短回答
max_new_tokens: 2048     # 标准回答
max_new_tokens: 8192     # 长文本生成
max_new_tokens: 32768    # 极长文本（需要模型支持）
```

### 7.4 常见问题与解决方案

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 训练 loss 不下降 | 学习率太小 | 增加 `learning_rate` |
| 训练 loss 震荡 | 学习率太大 | 减小 `learning_rate`，增加 `grad_clip` |
| 显存溢出 (OOM) | batch_size 太大 | 减小 `batch_size`，增加 `accumulation_steps` |
| 生成重复内容 | temperature 太低 | 增加 `temperature`，减小 `top_p` |
| 生成质量差 | 模型未充分训练 | 增加 `epochs`，检查数据质量 |
| 长文本性能差 | 超出训练长度 | 减小 `max_seq_len` 或外推微调 |
| DPO 后能力下降 | 学习率太大 | 减小 `learning_rate` 至 1e-9 |
