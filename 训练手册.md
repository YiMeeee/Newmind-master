# MiniMind 训练手册

本文档介绍 MiniMind 项目中各个训练脚本的用途和使用方法。

---

## 📁 训练脚本总览

| 脚本名 | 阶段 | 用途 | 显存需求 |
|--------|------|------|----------|
| `train_pretrain.py` | 阶段一 | 基座模型预训练 | 高 |
| `train_full_sft.py` | 阶段二 | 全量参数指令微调 | 高 |
| `train_lora.py` | 阶段二 | LoRA 参数高效微调 | 低 |
| `train_dpo.py` | 阶段三 | 直接偏好优化（对齐） | 中 |
| `train_distillation.py` | 独立 | 知识蒸馏（模型压缩） | 中 |
| `train_distill_reason.py` | 独立 | 推理能力蒸馏 | 中 |

---

## 🚀 标准训练流程

### 完整流程（从零开始）

```
预训练 → 指令微调 → 偏好对齐
   ↓          ↓          ↓
基础能力  指令遵循能力  人类价值观对齐
```

### 快捷流程（使用已有基座模型）

```
基座模型 → 指令微调 → 偏好对齐（可选）
```

---

## 📖 各阶段详解

### 1️⃣ 预训练 (`train_pretrain.py`)

**目标**：在大规模无标注语料上训练，让模型学习语言的基本规律和世界知识。

**数据格式**：纯文本语料（如维基百科、书籍、网页等）

**特点**：
- 无监督学习，不需要人工标注
- 训练时间最长，数据量最大
- 决定模型的基础能力上限

**启动命令**：
```bash
python trainer/train_pretrain.py
```

---

### 2️⃣ 指令微调 (`train_full_sft.py` / `train_lora.py`)

**目标**：让模型学会理解和遵循人类指令，从"续写文本"变成"完成任务"。

**数据格式**：指令 - 回答对（如：`{"instruction": "...", "output": "..."}`）

#### 两种微调方式对比：

| 方式 | 脚本 | 优点 | 缺点 |
|------|------|------|------|
| **全量微调** | `train_full_sft.py` | 效果好，充分适配任务 | 显存需求高，易过拟合 |
| **LoRA 微调** | `train_lora.py` | 显存低，训练快，可插拔 | 效果略低于全量 |

**建议**：
- 显存充足 + 追求效果 → 全量微调
- 显存有限 + 快速迭代 → LoRA

**启动命令**：
```bash
# 全量微调
python trainer/train_full_sft.py

# LoRA 微调
python trainer/train_lora.py
```

---

### 3️⃣ 偏好对齐 (`train_dpo.py`)

**目标**：让模型的输出更符合人类偏好和价值观（如：有用、诚实、无害）。

**数据格式**：偏好三元组（`prompt + chosen_response + rejected_response`）

**特点**：
- 替代传统的 RLHF（更简单、更稳定）
- 可选步骤，但能显著提升用户体验
- 需要人工标注的偏好数据

**启动命令**：
```bash
python trainer/train_dpo.py
```

---

### 4️⃣ 知识蒸馏 (`train_distillation.py`)

**目标**：用大模型（教师）指导小模型（学生）学习，在保持性能的同时减小模型体积。

**原理**：让学生模型模仿教师模型的输出分布（软标签）

**适用场景**：
- 模型部署到资源受限设备
- 需要更快的推理速度
- 已有大模型，想训练一个小模型继承其能力

**启动命令**：
```bash
python trainer/train_distillation.py
```

---

### 5️⃣ 推理蒸馏 (`train_distill_reason.py`)

**目标**：专门蒸馏模型的推理/思考能力，让模型学会逐步推理（CoT, Chain-of-Thought）。

**特点**：
- 数据中包含 `` 和 `<answer>` 标签
- 训练模型先"思考"再"回答"
- 显著提升数学、逻辑推理等任务的表现

**数据示例**：
```
用户：小明有 3 个苹果，吃了 1 个，又买了 2 个，现在有几个？

模型：<think>
首先，小明初始有 3 个苹果
吃了 1 个后：3 - 1 = 2 个
又买了 2 个后：2 + 2 = 4 个
</think>
<answer>4 个</answer>
```

**启动命令**：
```bash
python trainer/train_distill_reason.py
```

---

## 🔧 选择指南

| 你的需求 | 推荐脚本 |
|----------|----------|
| 从零训练一个大模型 | `pretrain` → `full_sft` → `dpo` |
| 微调已有的基座模型 | `full_sft` 或 `lora` |
| 显存不够，想微调 | `lora` |
| 让模型更符合人类偏好 | `dpo` |
| 压缩模型体积 | `distillation` |
| 增强推理/数学能力 | `distill_reason` |

---

## 📝 注意事项

1. **数据准备**：每个训练脚本需要对应的数据集格式，请参考 `dataset/` 目录下的数据处理代码
2. **超参数调整**：学习率、batch size 等参数需要根据你的硬件和数据集调整
3. **分布式训练**：所有脚本支持 DDP 分布式训练，多卡环境可加速训练
4. **断点续训**：建议定期保存 checkpoint，防止训练中断
